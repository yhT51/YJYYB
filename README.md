# 云端任务看板

## 📝 简介

这是一个基于 Supabase 云数据库的任务管理看板，支持多设备实时同步，数据永不丢失。

## ✨ 特性

- ☁️ **云端存储** - 所有数据保存在 Supabase 云数据库
- 🔄 **实时同步** - 多设备自动同步，随时随地访问
- 🎯 **任务管理** - 添加、编辑、删除、标记完成
- 🔍 **搜索过滤** - 支持关键词搜索和状态筛选
- 📅 **日期管理** - 设置任务日期和时间
- 🔥 **优先级** - 高、中、低三级优先级
- 💾 **自动保存** - 所有操作实时保存到云端

## 📦 使用说明

### 首次使用

1. 访问 [Supabase](https://supabase.com) 并注册账号
2. 创建新项目
3. 在项目设置中获取 API URL 和 Anon Key
4. 在任务看板配置页面输入这些信息
5. 在 Supabase 控制台创建 `tasks` 表（见下方 SQL 脚本）

### 创建数据表

在 Supabase SQL Editor 中执行以下脚本：

```sql
CREATE TABLE IF NOT EXISTS tasks (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  date TEXT,
  time TEXT,
  priority TEXT DEFAULT 'medium',
  done BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Enable read access for all users"
ON tasks FOR SELECT TO anon, authenticated USING (true);

CREATE POLICY "Enable insert access for all users"
ON tasks FOR INSERT TO anon, authenticated WITH CHECK (true);

CREATE POLICY "Enable update access for all users"
ON tasks FOR UPDATE TO anon, authenticated USING (true) WITH CHECK (true);

CREATE POLICY "Enable delete access for all users"
ON tasks FOR DELETE TO anon, authenticated USING (true);

CREATE INDEX IF NOT EXISTS idx_tasks_done ON tasks(done);
CREATE INDEX IF NOT EXISTS idx_tasks_created_at ON tasks(created_at DESC);
```

## 📱 功能说明

### 添加任务
1. 输入任务内容
2. 选择日期、时间（可选）
3. 选择优先级
4. 点击"添加任务"

### 编辑任务
1. 鼠标悬停在任务上
2. 点击✏️编辑按钮
3. 修改内容后保存

### 删除任务
1. 鼠标悬停在任务上
2. 点击🗑️删除按钮
3. 确认删除

### 搜索任务
- 在搜索框输入关键词
- 实时过滤显示匹配的任务

### 筛选任务
- 全部：显示所有任务
- 待完成：只显示未完成的任务
- 已完成：只显示已完成的任务
