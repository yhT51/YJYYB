# 如何切换 WorkBuddy 工作区 - 详细操作指南

## 前言

本指南将一步步教您如何将 WorkBuddy 的工作区从 `C:\Users\Administrator\WorkBuddy\Claw` 切换到 `D:\XLX\XLXWJ`。

---

## 第一步：找到您的 IDE

### 什么是 IDE？
IDE 是集成开发环境（Integrated Development Environment），就是您当前用来和我对话的这个软件界面。

### 如何确认您正在使用的 IDE？
观察您的界面，寻找以下线索：

#### Visual Studio Code (VS Code)
**识别特征：**
- 左侧有一个深色的侧边栏，有多个图标按钮
- 顶部有 `File`（文件）、`Edit`（编辑）、`View`（查看）等菜单
- 底部有蓝色或紫色的状态栏
- 窗口左上角显示 "Visual Studio Code"

#### JetBrains 系列 (如 IntelliJ IDEA, PyCharm, WebStorm)
**识别特征：**
- 左侧有项目目录树
- 顶部有菜单栏
- 右上角有运行、调试按钮
- 界面颜色通常是深色或浅色主题

#### JetBrains WorkBuddy 插件
**识别特征：**
- 右侧有一个 "WorkBuddy" 的面板或窗口
- 界面集成在 JetBrains IDE 中
- 可以看到和我对话的聊天窗口

---

## 第二步：在 VS Code 中切换工作区

如果您使用的是 **Visual Studio Code**，请按以下步骤操作：

### 方法一：通过菜单切换

1. **打开 VS Code**
   - 找到您当前正在使用的 VS Code 窗口

2. **打开文件菜单**
   - 点击窗口顶部的 `File`（文件）菜单
   - 或者按快捷键 `Ctrl + Shift + P`，输入 "Open Folder"

3. **打开新文件夹**
   - 选择 `Open Folder...`（打开文件夹...）
   - 在弹出的文件选择窗口中，找到 `D:` 盘
   - 依次进入：`D:` → `XLX` → `XLXWJ`
   - 点击 "选择文件夹" 或 "Select Folder"

4. **确认切换**
   - VS Code 会询问是否信任此文件夹中的作者
   - 点击 "Yes, I trust the authors"（是的，我信任这些作者）

### 方法二：通过快捷方式

1. 按 `Ctrl + K` 然后按 `Ctrl + O`
2. 在弹出的文件夹选择器中导航到 `D:\XLX\XLXWJ`
3. 点击"选择文件夹"

### 验证切换成功

切换成功后，您会看到：
- VS Code 左侧的文件资源管理器显示 `XLXWJ` 的内容
- 可以看到我们刚才复制过去的 `医嘱对接系统文档.md` 文件

---

## 第三步：在 JetBrains IDE 中切换工作区

如果您使用的是 JetBrains 系列 IDE（如 IntelliJ IDEA, PyCharm, WebStorm 等），请按以下步骤操作：

### 方法一：打开新项目

1. **启动 JetBrains IDE**
   - 打开您正在使用的 JetBrains IDE

2. **打开欢迎界面**
   - 如果当前打开了项目，点击 `File` → `Close Project`（关闭项目）
   - 等待进入欢迎界面

3. **打开新目录**
   - 在欢迎界面点击 `Open`（打开）
   - 或者点击 `File` → `Open...`
   - 在文件选择器中找到 `D:\XLX\XLXWJ`
   - 点击 `OK`

4. **信任项目**
   - 如果出现信任提示，选择 `Trust Project`（信任项目）

### 方法二：最近项目切换

1. 点击 `File` → `Open Recent`（最近打开）
2. 如果 `D:\XLX\XLXWJ` 已经在列表中，直接点击
3. 如果不在列表中，选择 `Open...` 然后选择该目录

### 验证切换成功

切换成功后，您会看到：
- 左侧项目视图显示 `XLXWJ` 目录
- 可以看到 `医嘱对接系统文档.md` 文件

---

## 第四步：重新启动 WorkBuddy

切换工作区后，需要重新启动 WorkBuddy：

### 如果 WorkBuddy 是独立运行的应用

1. **关闭当前 WorkBuddy**
   - 找到 WorkBuddy 的窗口或托盘图标
   - 点击关闭或退出

2. **重新启动 WorkBuddy**
   - 双击 WorkBuddy 图标启动
   - WorkBuddy 会自动检测新打开的工作区

### 如果 WorkBuddy 是 IDE 插件

1. **刷新插件**
   - 在 IDE 中，找到 WorkBuddy 插件面板
   - 点击刷新按钮（通常是一个循环箭头图标）
   - 或者重启 IDE（`File` → `Exit`，然后重新打开）

2. **确认连接**
   - WorkBuddy 应该自动连接到新的工作区
   - 可以看到聊天窗口重新打开

---

## 第五步：验证切换成功

完成上述步骤后，请验证：

```

```

### 2. 与我对话确认
发送一条消息给我，例如：

## 常见问题解答

### Q1: 我找不到 IDE 怎么办？
**A:** 请查看您的任务栏或桌面，找到您正在运行的应用程序窗口。如果不确定，可以截图发给我，我帮您识别。

### Q2: 切换工作区会丢失数据吗？
**A:** 不会！文件复制是独立操作，原文件仍然在 `C:\Users\Administrator\WorkBuddy\Claw` 中，新文件在 `D:\XLX\XLXWJ` 中，两个地方都有备份。

### Q3: 我可以同时打开多个工作区吗？
**A:** 可以！VS Code 支持打开多个文件夹（`File` → `Add Folder to Workspace`），但 WorkBuddy 通常只绑定一个主工作区。

### Q4: 切换后，我之前的对话记录还有吗？
**A:** 工作区切换不会删除对话记录。WorkBuddy 的对话记录通常保存在应用数据目录中，与工作区无关。

### Q5: 我不记得使用了哪个 IDE，怎么办？
**A:** 尝试以下方法：
- 查看任务栏的应用图标
- 按 `Alt + Tab` 查看打开的应用
- 截图给我，我帮您识别
